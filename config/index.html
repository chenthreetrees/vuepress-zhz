# 配置

该项目通过配置文件来控制页面的显示，避免编码过程中冗余的页面组件配置代码 配置文件的结构如下：

const config = {
  requestUrl:{},
  listQuery:{},
  searchOptions:{},
  menuOptions:{},
  tableOptions:{},
  formOptions:{},
}
export default config

# requestUrl

页面数据增删改查的url，结构如下：

requestUrl: {
  loadUrl: '',
  insertUrl: '',
  updateUrl: '',
  deleteUrl: '',
  importUrl:''
}
  • loadUrl
    ZaTable组件加载数据时调用
  • insertUrl
    FormDialog添加数据时调用
  • updateUrl
    FormDialog更新数据时调用
  • deleteUrl
    ZaTable组件删除数据时调用
  • importUrl
    ImportDialog组件导入数据时调用

参考

这里封装了数据的通用请求，如果网络请求逻辑不一致，可以自行修改tableRequest文件 参考：tableRequest.js

# listQuery

ZaTable组件加载数据时需要的请求参数，可与ZaSearchBar的搜索条件配合，结构如下：

listQuery:{}

参考

考虑到分页功能，ZaTable组件在加载数据的时候，listQuery会混入page和size两个默认参数，如果网络请求逻辑不一致，可以自行修改tableRequest.js文件

# searchOptions

ZaSearchBar组件的配置信息，结构如下：

searchOptions：{
  span: 6,
  showCount: 4,
  items:[]
}

# span

  • 类型：
    number
  • 默认值：
    -
  • 说明： 采用24栅格布局，span的值将影响每一个item的宽度比例 例如：span为6，则每一行将排列4个item

# showCount

  • 类型：
    number
  • 默认值：
    -
  • 说明： 表示第一行显示多少个item，当items超过showCount时，将显示下拉按钮

# items

  • 类型：
    Array
  • 默认值：
    -
  • item公共属性：
属性 类型
默认
 说明
template string — item的类型：select，date-picker，click，不设置显示为input输入框
span number — 单独为item设置span
prop string — 对应searchData的属性
placeholder string — 输入框默认显示的文字
queryKey string,string数组 — 对应listQuery对象的属性，不设置则取prop

# select

 说明

valueKey string - 作为 value 唯一标识的键名，绑定值为对象类型时必填,确保唯一性

optionName string prop的值 extendData里面对应的候选项
labelKey string label 显示在输入框所对应的属性

# tr

  • 类型：
    Array
  • 默认值：
    -
  • item属性：
属性 类型
默认
 说明
show boolean — 该列是否显示
id number或string — 该列的唯一标识
prop string — 数据值属性,template时为slot的名称
label string — 列名
sortable boolean true 是否排序
fixed string — 固定列，right，left
width string — 列宽
minWidth string 120 列最小宽度
template boolean — 设置为true时，自定义列，使用slot
formatter function null 转换列的显示内容，function自带了整行的数据，参数为row

template示例：

{
  id: '2',
  label: '设备名称',
  prop: 'deviceName',
  show: true,
  template: true
}

{
  id: '3',
  label: '设备编号',
  prop: 'deviceNo',
  show: true,
  template: true
}

  <BaseTable>
    <template v-slot:deviceName="{obj}">
      {{ obj.row.deviceName }}
    </template>
    <template v-slot:deviceNo="{obj}">
      {{ obj.row.deviceNo }}
    </template>
  </BaseTable>

formatter示例：

export function dateformat(dataStr, pattern = 'YYYY-MM-DD') {
  return moment(dataStr).format(pattern)
}

//配置item
{
  id: '5',
  label: '检定时间',
  prop: 'checkTime',
  show: true,
  formatter: function(row) {
    return row ? dateformat(row.checkTime) : ''
  }
}

# rules

  • 类型：
    Object
  • 默认值：
    -
  • 说明：
    表单验证规则

示例：

export function validateEmpty(value, trigger = 'blur') {
  return {
    required: true,
    message: value + '不能为空',
    trigger: trigger
  }
}

rules: {
  deviceName: [validateEmpty('设备名称', 'change')],
  saveAddress: [validateEmpty('设备保管地址')]
}

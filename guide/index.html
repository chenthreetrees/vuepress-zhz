# 介绍

vue-zhz-admin 是一个后台前端解决方案，它基于 vue 和 element-ui实现。它使用了最新的前端技术栈，动态路由，权限验证，提炼了典型的业务模型，提供了丰富的功能组件和可拓展的配置，它可以帮助你快速搭建企业级后台前端页面。

参考

使用本项目要求对vue生态有最基本的了解 本项目提供简单的框架，适合进行二次开发，可以参考基于此框架开发的平台

# 快速上手

# 安装cli
npm install zhz-cli -g

# 克隆模板代码
zhz init zhz-admin [your project name]

# 进入项目目录
cd [your project name]

# 建议不要用 cnpm 安装 会有各种诡异的bug 可以通过如下操作解决 npm 下载速度慢的问题
npm install --registry=https://registry.npm.taobao.org

# 本地开发 启动项目
npm run dev

# 路由和侧边栏

在 @/router/index.js 文件里配置对应的路由，侧边栏就能动态生成。

# 配置

//当设置 true 的时候该路由不会在侧边栏出现 如401，login等页面，或者如一些编辑页面/edit/1
hidden: true // (默认 false)

//当你一个路由下面的 children 声明的路由大于1个时，自动会变成嵌套的模式--如组件页面
//只有一个时，会将那个子路由当做根路由显示在侧边栏--如引导页面
//若你想不管路由下面的 children 声明的个数都显示你的根路由
//你可以设置 alwaysShow: true，这样它就会忽略之前定义的规则，一直显示根路由
alwaysShow: true

name: 'router-name' //设定路由的名字，必填，与页面的name一致，否则tagsView的缓存失效
meta: {
  title: 'title' //设置该路由在侧边栏和面包屑中展示的名字
  icon: 'svg-name' //设置该路由的图标
  noCache: true //如果设置为false，则tagsview在切换的时候，页面将被缓存,
  perms:[] //参考权限
}

示例：

  {
    path: '/test',
    component: Layout,
    alwaysShow: true,
    meta: {
      title: '测试',
      icon: 'test'
    },
    children: [{
      path: 'test',
      name: 'Test',
      component: () => import('@/views/test'),
      meta: {
        title: '测试页面',
        noCache: true,
        perms: ['POST /admin/list']
      }
    }]
  },

路由

路由分为两种，constantRoutes 和 asyncRoutes constantRoutes： 代表那些不需要动态判断权限的路由，如登录页、404、等通用页面 asyncRoutes： 代表那些需求动态判断权限并通过 addRoutes 动态添加的页面

# 权限

权限规则需与后台约定好，在登录成功获取用户信息的时候，后台返回权限列表，可以用string数组表示， 本项目默认['*']表示拥有所有权限，以下几种权限的应用示例：

路由权限：

// 用户信息的权限列表只有包含`POST /admin/list`才会在侧边栏生成页面入口
meta: {
  title: '测试页面',
  noCache: true,
  perms: ['POST /admin/list']
}

操作权限：

// 用户信息的权限列表只有包含`POST /admin/dv/device/delete`才会显示删除按钮
menuOptions: {
    hasDelete: true,
    hasCreate: true,
    deletePermission: ['POST /admin/dv/device/delete']
  }

// table的操作列的按钮权限
operation: {
    label: '操作',
    data: [
      {
        label: '修改',
        id: '2',
        permission: ['POST /admin/dv/device/insertOrUpdate']
      }
    ]
  }
